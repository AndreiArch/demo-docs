  ## DairyComp 305 - Version 25.11

  **Release Date:** December 2, 2025

  DC305 version 25.11 contains various bug fixes, customer-requested enhancements, and integration improvements for our integration partners, as well as many other improvements.

  Expand the topics below to learn more about the updates in this release:

  <ExpandableHeading>
    ## Base Program

    - DC305 can now capture event entry dates and times. Use the following new items to track event entry (version 10 cowfiles only):
      - Item type 175: A calculated item that extracts the date the event was entered into DC305. This differs from the actual event date, because events are often recorded on a date that is not the same as when they occurred for the animal.
      - Item type 176: A calculated item that extracts the time the event was entered into DC305. This is the time the event was entered into DC305, not the time the event occurred for the animal.
      - Event entry date and time can be see in the event editor for each event on the CowCard. This field is read only. Events that existed prior to this release show the event date and `00:00` for the time.
        :inlineImage[]{src="https://api.archbee.com/api/optimize/3h8pSJcGUXEi4HGCCYPTA/woOINZNVH_aJ50RsHIBHP_2511-event-editor.png" alt caption}
    - DC305 now supports additional item details:
      - Item type 124 now has a new column that can be used to show total lactation production components (i.e., the sum of fat and protein). Choose **Fat+Ptn** for the column to see total lactation production components:
        :inlineImage[]{src="https://api.archbee.com/api/optimize/3h8pSJcGUXEi4HGCCYPTA/cL9kkDV9sycCzS0uGN8sh_2511-item124.png" alt caption}
      - Item type 125 now has additional Operand 2 options that can be used to show total lactation as a days in milk (DIM) value for fat, protein, and the sum of fat and protein.
        For example, `PRT30` with `Operand 1:30` is the total lactation protein as of DIM 30:
        :inlineImage[]{src="https://api.archbee.com/api/optimize/3h8pSJcGUXEi4HGCCYPTA/tcgCUwmouIA2zSjCHCJEl_2511-item125props.png" alt caption}
      - Item type 126 now has additional Operand 2 options that can be used to show total lactation at the time of a specific date item for fat, protein, and the sum of fat and protein.
        For example, `POCOM` with `Item Name: PODAT` and `Operand 2: Fat+Ptn` returns the sum of fat and protein at the time of the PODAT (preg/open date):
        :inlineImage[]{src="https://api.archbee.com/api/optimize/3h8pSJcGUXEi4HGCCYPTA/1xmHbdJXFNUu6N8FJ6KG7_2511-item126.png" alt caption}
    - Resolved an issue that caused the DC Client task table to be overwritten by the DC Server.
    - Updated the 646 stud code to show as sexed semen.
    - Russian language vet codes are now supported in the [ALTER\5](../../ReferenceGuide/ALTER/Alter5-VetList.htm) function.
    - [Integrated help](../../GetStarted/in-app-help.htm) has been added to the **SERVER** button when there are [HELD commands](../../FAQs/ResolveHeld.htm).
    - The `SAVE\2` option in [SETUP](../../ReferenceGuide/Config/SetupCmdIntro.htm) now allows you to set the number of backups to keep.
    - When the `\GG` switch is used, item type 54 in the [EVENTS](../../ReferenceGuide/Events/EventCommand.htm) function now displays the item data for the corresponding event.
    - The PLOT function now sets the minimum y-axis value for Somatic Cell Count (SCC) charts to 50 (was 0). This improves the visual contrast and readability of stacked multi-bar charts involving SCC.
      :inlineImage[]{src="https://api.archbee.com/api/optimize/3h8pSJcGUXEi4HGCCYPTA/lD8-qe9MCJkFZGrndXO35_2511-scc-plot.png" alt caption}
    - Item type 130 in the [EVENTS](../../ReferenceGuide/Events/EventCommand.htm) function now displays correctly.
    - When the `\GG` switch is used, item type 54 in the [EVENTS](../../ReferenceGuide/Events/EventCommand.htm) function now displays the item data for the corresponding event.
    - A default printer is no longer required. The following message now displays:
      Printing is disabled because no default printer has been found. To stop seeing this message, add a default printer or disable printing:
      **ADD A PRINTER:** In Windows Settings, add or select a default printer. Restart DairyComp and clear the message. Enter SETUP in the command line. In the Printer tab, check the box next to Print Enabled. Click OK to save your changes.
      **DISABLE PRINTING:** Enter SETUP in the command line. In the Printer tab, check the box next to Disable All Printing. Click OK to save your changes.
    - Resolved an issue with the event translation file when it was automatically created.
    - Resolved an issue with a corrupted DCGUIDE file.
    - Resolved an issue causing license errors for Russian customers.

    ## New Feature! CowVal 1.5

    This DC305 update delivers some exciting enhancements to the CowVal feature!

    :inlineImage[]{src="https://api.archbee.com/api/optimize/3h8pSJcGUXEi4HGCCYPTA/v4MfPXgq8t9v87Rc6CzW5_2080-cowval15.png" alt caption}For herds that are cross-bred or that contain multiple breeds, we are aware that it was difficult to base a CowVal score on quantity alone. With the CowVal 1.5 update, you'll continue to see details on the quantity of the milk produced, but now when you run COWVAL\F, you'll also receive information on **Milk Fat** and **Milk Protein**.

    Adjusting the current CowVal formula to include Protein and Fat benefits farms that would like to use COWVAL to set a score on their cows, as these components are key values in the milk check.

    Read below to learn :Link[what changed]{href newTab="true"} and :Link[how to set up CowVal 1.5]{href newTab="true"}.

    ### What Changed?

    CowVal 1.5 takes into account the animal's contribution of **Milk Fat** and **Milk Protein**. As you can see in the example to the right, we have added the **Protein Price** and **Fat Price** to give a value of the components that the animal is contributing. We also have added fields for **Lactation Fat** and **Lactation Protein** values to serve as defaults if a cow doesn’t have test day data. The **Mature Equivalent** values are based off of data from your herd; with the use of this formula, we can better compare animals in a multi-breed herd.

    ### How to Set Up CowVal 1.5

    To use the new CowVal 1.5 feature, enter the command COWVAL\F and then select **Edit Parameters** to access the updated table. The items used are still CWVAL and PGVAL and will be populated with the new formula. To schedule the recalculation of these items, you can use the command **COWVAL\FG**.
