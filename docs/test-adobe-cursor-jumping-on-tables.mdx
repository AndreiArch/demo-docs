## Exercise 1.2: AI-Generated Metadata and Smart Crops

**Duration:** 6 minutes

**Understanding AI-Powered Asset Intelligence**

When assets arrive in AEM Assets as a Cloud Service, Adobe AI immediately goes to work. It generates three distinct metadata fields automatically including a **Generated Title**, a **Generated Description**, and **Generated Keywords&#x20;**&#x77;hich appear in a dedicated **AI Generated tab** on the asset's Properties page. These are separate from any manually entered metadata; if no manual title is defined, AEM uses the AI-generated title by default across card and list views.

Beyond metadata and tagging, **Smart Crop** which is powered by Adobe Sensei via Dynamic Media identifies the focal point of each image and pre-generates renditions for common aspect ratios automatically. This means channel-ready variations are available without generating separate stored files for each format.&#x20;

**Value for Your Organization:**

- AI-generated metadata provide broader taxonomy coverage for discovery and filtering
- Smart Crops eliminate repetitive production work for common channel formats
- Assets are immediately usable across channels without designer intervention

**Understanding Intelligent Asset Discovery**

AEM Assets provides multiple AI-powered search modes. **Omnisearch** (the standard search bar) factors Smart Tags and AI-generated metadata fields into results alongside standard metadata. **Contextual Search** allows users to type natural language text prompts and AEM automatically converts them into search filters. Search results that match all terms in metadata fields display first, followed by results that match terms in Smart Tags only.

**Value for Your Organization:**

- Contextual Search converts natural language prompts into search filters automatically
- Global teams can search in their own language and terminology
- Semantic search surfaces relevant content even without exact keyword matches

**Understanding Collections in AEM Assets**

Collections allow teams to organize and share curated sets of assets without moving files or disrupting folder governance. Unlike folders, a collection can include assets from different locations across the DAM. - Smart Collections automatically surface the latest assets matching defined criteria

## Exercise 1.6: Asset Approval for Distribution

**Duration:** 3 minutes

**Understanding Approval Status as Governance**

Not all assets in the repository are ready for broad distribution. In AEM Assets as a Cloud Service, each asset can be assigned one of three review states via the **Status** dropdown in the Basic tab of asset Properties: **Approved**, **Rejected**, or **No Status** (the default). When setting an asset to Approved, an **Approval Target** field appears, allowing you to direct the asset to Content Hub, Delivery (Dynamic Media with OpenAPI), or both, depending on your environment.

**Value for Your Organization:**

- Precise governance over which assets reach external distribution channels
- Approval Target field directs assets to the correct downstream channel
- Visual indicators make approval status immediately clear
